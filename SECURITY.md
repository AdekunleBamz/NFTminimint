# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 2.x.x   | :white_check_mark: |
| 1.x.x   | :x:                |

## Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly:

1. **Do NOT** open a public issue
2. Email security concerns to the maintainer
3. Include detailed steps to reproduce
4. Allow 48 hours for initial response

## Security Best Practices

When deploying these contracts:

- Always verify contract code on block explorers
- Use hardware wallets for owner accounts
- Test thoroughly on testnets first
- Keep private keys secure and never commit them
- Use multi-sig for high-value operations

## Audit Status

These contracts have not been formally audited. Use at your own risk.
